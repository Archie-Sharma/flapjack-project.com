If you'd like to contribute packages for other systems, please check out the %a{:href=>"http://wiki.flapjack-project.com/packaging_guide"} packaging guide! %div#running %h2 Architecture %p %em Flapjack consists of several components that  %a{:href => "http://en.wikipedia.org/wiki/Unix_philosophy"}> do one thing and do it well \: %img{:src => "images/flapjack-architecture.png", :class => 'architecture', :alt => 'architecture diagram'} %ul %li %code flapjack-worker executes the checks and reports back results. %li %code flapjack-notifier notifies people if the check results are bad. %li %code flapjack-admin lets you configure checks through a web interface. %li %code beanstalkd is a work queue used to distribute checks and results. %li %code CouchDB is a document-oriented database that checks and events are stored in. %p Each component is loosely coupled, which makes swapping out one part of the system for another very easy. Flapjack uses well defined and tested APIs between the components to guarantee consistency. %p This also makes it very easy to extend Flapjack by writing custom admin interfaces or check persistence backends. %p Find out more about %a{:href=>"http://wiki.flapjack-project.com/architecture"} Flapjack's architecture. %div#administering %h2 Administering %p Grab the %a{:href => "http://github.com/auxesis/flapjack-admin/tree/master#readme"} flapjack-admin web interface to start configuring checks, then use the %code flapjack-populator included with %code flapjack-admin to start running the checks: %blockquote.commands %pre :preserve $ cd path/to/flapjack-admin $ bin/flapjack-populator %div#help %h2 Getting help %p Check out the %a{:href=>"http://wiki.flapjack-project.com"} wiki for extended documentation on installing, using, packaging, and developing Flapjack. %div#mailinglist %img{ :src => "http://groups.google.com/intl/en/images/logos/groups_logo_sm.gif", :height => "30", :alt => "Google Groups", :width => "140" } Sign up to the mailing list. %form{:action => "http://groups.google.com/group/flapjack-project/boxsubscribe"} Email: %input{ :name => "email", :type => "text" } %input{ :name => "sub", :type => "submit", :value => "Subscribe" } %a{:href => "http://groups.google.com/group/flapjack-project/topics"} View the archives. %div#who %h2 Who's reponsibile?
